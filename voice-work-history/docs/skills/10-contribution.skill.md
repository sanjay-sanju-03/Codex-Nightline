# Contribution Rules

Before adding a feature, ask:

1. Does it reduce worker effort?
2. Does it preserve worker ownership and review?
3. Does it fit a local-first MVP?
4. Can it be demonstrated reliably in the time available?

If the answer is no, do not add it during the hackathon.

## Engineering Rules

- Keep API keys server-only.
- Validate external data at boundaries.
- Prefer small, testable changes.
- Update the relevant skill and product docs with behavior changes.
